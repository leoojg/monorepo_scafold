# Design: Internacionalização (i18n) da Plataforma Admin

**Data:** 2026-03-08
**Idiomas:** Inglês (padrão) + Português (pt-BR)
**Escopo:** Frontend (admin) + ajuste mínimo no backend (error codes)

---

## 1. Arquitetura Geral

### Dependências

**Frontend (apps/admin):**

- `i18next` — core de internacionalização
- `react-i18next` — bindings React (hook `useTranslation`)
- `i18next-browser-languagedetector` — detecção automática de idioma
- `DropdownMenu` do shadcn/ui — componente para o menu do usuário

### Idioma Padrão e Detecção

- Idioma padrão: `en`
- Detecção automática: `i18next-browser-languagedetector` checa `localStorage` primeiro, depois `navigator.language`
- Se `navigator.language` for `pt-BR`, troca automaticamente
- Persistência de preferência: `localStorage`
- Fallback: `en`

## 2. Estrutura de Traduções

### Organização por Namespaces

```
apps/admin/src/i18n/
├── index.ts              # configuração do i18next
├── locales/
│   ├── en/
│   │   ├── common.json   # botões, nav, status, labels genéricos, erros
│   │   ├── auth.json     # login
│   │   ├── tenants.json
│   │   ├── companies.json
│   │   ├── users.json
│   │   └── activity.json
│   └── pt-BR/
│       ├── common.json
│       ├── auth.json
│       ├── tenants.json
│       ├── companies.json
│       ├── users.json
│       └── activity.json
```

### Convenção de Chaves

Chaves nested dentro de cada namespace:

```json
// tenants.json (en)
{
  "list": {
    "title": "Tenants",
    "search": "Search tenants..."
  },
  "form": {
    "name": "Name",
    "slug": "Slug"
  }
}
```

```json
// tenants.json (pt-BR)
{
  "list": {
    "title": "Organizações",
    "search": "Buscar organizações..."
  },
  "form": {
    "name": "Nome",
    "slug": "Slug"
  }
}
```

### Uso nos Componentes

```tsx
const { t } = useTranslation('tenants');
t('list.title'); // → "Tenants" ou "Organizações"
```

### Tradução de "Tenant"

- en: "Tenant" / "Tenants"
- pt-BR: "Organização" / "Organizações"

## 3. Componente UserMenu

Dropdown no header (lado direito) usando `DropdownMenu` do shadcn/ui:

- Exibe o nome do usuário
- Ao clicar, abre dropdown com:
  - **Seletor de idioma** — ícone de globo + idioma atual. Opções: "English" / "Português (BR)"
  - **Separador**
  - **Logoff** — ícone de logout + "Sign out" / "Sair"

### Comportamento

- Trocar idioma chama `i18n.changeLanguage()`
- `i18next-browser-languagedetector` persiste automaticamente em `localStorage`
- A UI re-renderiza instantaneamente (react-i18next cuida disso)
- Logoff usa o `logout()` do `AuthProvider` existente

## 4. Tratamento de Erros da API

### Backend — Error Codes

A API adiciona um campo `errorCode` nas respostas de erro:

```typescript
// error-codes.enum.ts
export enum ErrorCode {
  AUTH_INVALID_CREDENTIALS = 'AUTH_INVALID_CREDENTIALS',
  TENANT_NOT_FOUND = 'TENANT_NOT_FOUND',
  COMPANY_NOT_FOUND = 'COMPANY_NOT_FOUND',
  USER_NOT_FOUND = 'USER_NOT_FOUND',
  AUDIT_LOG_NOT_FOUND = 'AUDIT_LOG_NOT_FOUND',
  VALIDATION_ERROR = 'VALIDATION_ERROR',
}
```

O `HttpExceptionFilter` inclui `errorCode` no response JSON:

```json
{
  "statusCode": 401,
  "errorCode": "AUTH_INVALID_CREDENTIALS",
  "message": "Invalid credentials",
  "timestamp": "2026-03-08T..."
}
```

Error codes documentados no Swagger via `@ApiResponse` → Orval gera tipos automaticamente no frontend.

### Frontend — Mapeamento Automático

O `errorCode` da API é usado diretamente como chave de tradução:

```json
// common.json (en)
{
  "errors": {
    "AUTH_INVALID_CREDENTIALS": "Invalid credentials",
    "TENANT_NOT_FOUND": "Tenant not found",
    "UNKNOWN": "An unexpected error occurred"
  }
}
```

Interceptor no Axios:

```typescript
const errorCode = error.response?.data?.errorCode;
const message = i18n.exists(`common:errors.${errorCode}`)
  ? i18n.t(`common:errors.${errorCode}`)
  : i18n.t('common:errors.UNKNOWN');
```

## 5. Formatação de Datas e Números

Usa APIs nativas do browser configuradas pelo locale ativo do i18next:

- `Intl.DateTimeFormat` para datas
- `Intl.NumberFormat` para números

Exemplos:

| Formato | en           | pt-BR        |
|---------|--------------|--------------|
| Data    | 03/08/2026   | 08/03/2026   |
| Número  | 1,234.56     | 1.234,56     |

Helpers utilitários que leem o locale do i18next:

```typescript
export function formatDate(date: Date): string {
  return new Intl.DateTimeFormat(i18n.language).format(date);
}

export function formatNumber(value: number): string {
  return new Intl.NumberFormat(i18n.language).format(value);
}
```
